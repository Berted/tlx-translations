Description

In a 1 dimensional coordinate system, Feli has a rabbit named Rbit. Rbit likes to jump a lot. In every jump, Rbit can move to the right or move to the left.

Every time Rbit jumps, the jump distance is reduced by 1, if compared to the previous one. If Rbit jumps a 7 long-distance from coordinate 3 to 10, then in the next one it jumps for 6 long-distance ends up in coordinate 4 or 16. As Rbit loves jumping, Rbit will keep jumping until it's jumping distance reach 0. Rbit can also jump into a negative coordinate point.

Rbit lives at coordinate point 0 and wants to go outside. The only way out is at the coordinate point $$$N$$$. Help Rbit to get outside by specifying the minimum first jumping distance and the direction of every jump so that Rbit stops right at coordinate $$$N$$$. If Rbit can't go outside output "-1" instead.

Input Format

Input starts with single integer $$$T$$$, the number of test cases in the input. The next $$$T$$$ lines contain a single integer $$$N$$$ with $$$N$$$ is the coordinate position to be achieved.

Output Format

For each test case, if Rbit can go outside output "Case #$$$X$$$: $$$Y$$$ $$$S$$$" where $$$X$$$ is the case number (starting from 1). $$$Y$$$ is the minimum jumping distance of Rbit, and $$$S$$$ is a $$$Y$$$ length string that tells the direction of every jump ($$$Si$$$ contains "+" if Rbit jumps to the right at $$$i$$$-th jump or "-" if-else). If there're more than one way to achieve the goal, output one of it. If Rbit can't get outside output "Case #$$$X$$$: -1" instead.

Constraints

  • $$$1 ≤ T ≤ 100$$$.
  • $$$1 ≤ N ≤ 106$$$.

Sample Input

3
1
3
4

Sample Output

Case #1: 1 +
Case #2: 2 ++
Case #3: 3 ++-

Explanation

In the third case, Rbit can jump 3 to the right, 2 to the right and lastly 1 to the left to reach position 4.
